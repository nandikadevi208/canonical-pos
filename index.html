Introduction

Why canonical forms matter

Any digital logic function can be written many different ways — but two forms are unique and standard: canonical Sum of Products (SOP) and canonical Product of Sums (POS). This site treats every conversion as a small proof: we never just hand you the final expression — we build it, one Boolean identity at a time.

Boolean Algebra Basics

Variables, literals & operators

A variable (A, B, C…) holds 0 or 1. A literal is a variable or its complement (A or A'). Three operators combine them: AND (·), OR (+), NOT (').

Identities used throughout this site

ComplementY + Y' = 1  ·  Y·Y' = 0Introducing a missing variable without changing value

Canonical Sum of Products

Definition

A Sum of Products (SOP) expression is an OR of AND terms, e.g. AB' + AC. A canonical SOP is stricter: every product term — called a minterm — must contain every variable of the function exactly once, complemented or not.

Standard SOP vs Canonical SOP

Standard SOPCanonical SOP
Product terms may omit variablesEvery product term contains all variables
Usually shorter / simplifiedLonger, unique form (one per function)

Minterms

Each minterm evaluates to 1 for exactly one input combination and 0 everywhere else. For a row where variable X = 1, the minterm rule writes X plain; where X = 0, it writes X'. OR-ing all minterms of the 1-rows together reproduces the function exactly.

Variable value in one-rowLiteral in minterm
X = 1X (plain)
X = 0X' (complemented)

Shorthand: F = Σ m(list of one-row indices) — "the sum of minterms."

SOP Simplification

K-map grouping combines adjacent 1-cells (groups of 1, 2, 4, 8…) into fewer, shorter product terms — this is exactly what the automates using the Quine–McCluskey method.

Advantages

  • Unique, unambiguous representation
  • Directly derivable from a truth table
  • Natural starting point for AND-OR (or NAND-NAND) circuit design
  • Efficient when the function is mostly 0s (few one-rows)

Disadvantages

  • Can be long for many variables
  • Rarely the minimal circuit — needs simplification
  • Grows exponentially with variable count

Applications

  • Designing combinational circuits with AND-OR or NAND gate networks
  • Deriving minimal SOP for PLD/FPGA logic blocks
  • Foundation for K-map and Quine–McCluskey minimization
Canonical Product of Sums

Definition

A Product of Sums (POS) expression is an AND of OR terms, e.g. (A+B)(A'+C). A canonical POS is stricter: every sum term — called a maxterm — must contain every variable of the function exactly once, complemented or not.

Standard POS vs Canonical POS

Standard POSCanonical POS
Sum terms may omit variablesEvery sum term contains all variables
Usually shorter / simplifiedLonger, unique form (one per function)

Maxterms

Each maxterm evaluates to 0 for exactly one input combination and 1 everywhere else. For a zero-row where variable X = 1, the maxterm rule writes X'; where X = 0, it writes X plain. AND-ing all maxterms of the 0-rows together reproduces the function exactly.

Variable value in zero-rowLiteral in maxterm
X = 1X' (complemented)
X = 0X (plain)

Shorthand: F = Π M(list of zero-row indices) — "the product of maxterms."

About

About SOP·POS Verse

A single-file, self-contained learning tool for Canonical SOP and Canonical POS: mirrored theory pages, a truth-table generator, a real Quine–McCluskey K-Map solver (grouping both 1s and 0s), a calculator that shows full classroom working, 25 solved examples, and a 30-question quiz.

How the classroom-style steps work

Every conversion — in the calculator and in the solved examples — runs through a real algebra engine: it finds each term's missing variables, applies X = X(Y+Y') for SOP or X = X+YY' for POS, expands with the distributive law, and removes duplicate terms at the end. Nothing is hardcoded text — it's generated from your input.
