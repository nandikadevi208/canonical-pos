Theory

Boolean Algebra & Product of Sums

Boolean algebra works with two values, 0 and 1, combined using AND (·), OR (+) and NOT ('). Any digital logic function can be expressed purely in terms of these operators — and two standard ways of doing that are Sum of Products (SOP) and Product of Sums (POS).

Product of Sums (POS)

A POS expression is an AND of OR terms, e.g. (A+B)·(A'+C). The terms don't need to contain every variable — that's what separates standard POS from canonical POS.

Canonical Product of Sums

A canonical POS is stricter: every sum term — called a maxterm — must contain every variable of the function exactly once, complemented or not.

Variable value in zero-rowLiteral written in maxterm
X = 1X (complemented)
X = 0X (plain)

Shorthand: F = Π M(list of zero-row indices) — "the product of maxterms."

Maxterms

Each maxterm evaluates to 0 for exactly one input combination (the row it came from) and 1 everywhere else. AND-ing all of them together forces the function to 0 only where the truth table says it should be — nowhere else.

Advantages

  • Unique, unambiguous representation of any function
  • Directly derivable from a truth table
  • Convenient starting point for K-map simplification
  • Efficient when the function is mostly 1s (few zero rows)

Disadvantages

  • Can be long and hard to read for many variables
  • Rarely the most efficient circuit — needs simplification
  • Grows exponentially with variable count

Applications in Digital Electronics

  • Designing combinational logic circuits from specifications
  • Deriving minimal NOR-gate implementations (POS maps naturally to NOR logic)
  • Verifying and documenting logic behind PLDs / FPGAs
  • Teaching foundation for Karnaugh maps and Quine–McCluskey minimization
